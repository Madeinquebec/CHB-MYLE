HOW TO DOCUMENT ALERTS AND SPECIAL NEEDS:

  • “Alert” [A] is the equivalent of the big pink sticker on the top of the Patient File.
  • The Alert creation/documentation is according to the clinician judgement.
  • Do not add Alert if this is not really important. When there are too many alerts, people tend not to read them.
  • There is two different kind of Popup Window that can appear when you enter a Patient File: “Memo” and “Alert”.
  • The “Alert” Popup Window [A] appear when we enter a Patient File that is having an Alert documented.
  • The “Alert” Popup Window [A] is seen only by people that have a Clinical Account.
  • All relevant information needs to be documented in the “Alert” field [B] because it’s this content that appear in the Alert Popup Window [A], in the Patient Summary and in the Alert List (left margin) [C].

IMPORTANT

  • Here’s a list of the kind of information that is expected to be document in the “Alert and Special Need”:
    • All important disabilities
    • Do Not Resuscitate
    • AV Fistula Left/Right Arm
    • On Dialysis (precise if it’s Peritoneal / Hemo)
    • Lead pellets in situ: MRI contraindication
    • Narcotic Alert
    • Pregnancy EDC (EDC + 1 month)
    • Transplant patient
    • Unilingual Cree
    • Name Alert (added it to Alert and create Memo)
    • Patient difficult to reach
    • Fall Risk
    • Disability Equipment (wheelchair, walker, cane)
    • Low vision / Deafness
    • Homecare Patient
  • When the level of care is “Do Not Resuscitate (DNR)”, make sure you add it as an Alert and as an item in the Preventive Activities for the details.

Forms

Purpose: To fill Electronic Forms directly in MYLE.

When a user clicks on "Form [+]":

  • The "New Form" Window will display the forms "Recently used in this file".
  • With the forms "Recently used in this file", it's possible to :
    • "Duplicate": MYLE will create a new instance of the previously filled form and the user can add new information.
    • "Continue": MYLE won't create a new instance of the previously filled form. The user will continue to enter new information in the same form that was previously filled.
    • It is important to understand "Continue" and "Duplicate" import in the form previous information entered in this form.
IMPORTANT:
  • For the Evolutive Form (ex: obstetrical follow-up), the user will click on "Continue". So, the form won't be duplicate and the user will add new information in a previously filled form.
  • It is not recommended to use "Continue" for the Non-Evolution Form.

Search Field:

  • By entering keyword, MYLE will return the official blank forms and those who were recently used in this file.

Favorite Form:

  • It is possible to have Favorite Form by clicking on the star.

Electronic Signature:

  • There is no electronic signature in the forms, but:
    • The note that will contain the form will be approve by the user, so it will be consider as the electronic signature of the form.
    • There is a field in the form where the user can enter his/her name.
IMPORTANT:
  • It's mandatory to each user to enter his/her name in the appropriate field in the form.

Form Field Size:

  • The text field are limited to the space available in the form.

  • It's possible to add a sticky note to enter more information, but the content of this sticky note won't be print if the form is print. So, DO NOT USE sticky note.

Using the Pen Tool to draw :

  • MYLE allow the Pen Tool to draw in a form, but it needs to be used cautiously because it will be impossible to undo or to remove the draw without deleting the form and restarting all over again.

Form Content Modification:

  • As long a note is not approved, it's possible to enter in a form and make some change.
IMPORTANT:
  • Take note that it is necessary to click on the "Modify" Button in the note’s lower right corner before doing the changes. Otherwise, the modification won’t be saved.

Form Gearwheel:

  • The gearwheel will allow the two following actions:
    • "Delete": To remove permanently the form from the note.
    • "Share for future notes": Allow to transpose the form as well as the information added in it to the next clinical notes that will be opened in the file.
      • This is particularly useful for the follow-up forms, notably the obstetric forms.

VERY IMPORTANT:
Do not use it for now.
