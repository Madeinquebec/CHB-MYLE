Progress Note

IMPORTANT

Always enter a reason when creating a note because the content of the “Reason” field becomes the title for each note that appear in the list of notes [A]. So, it's easier to navigate the list of notes.

Purpose: To document the consultation reason.

[1] Free Text:

  • This is a free text field, but we ask to the clinician to use the content from the "Select from list" [2] as much as possible.

IMPORTANT:
  • If a note is written at a later date than the consultation date, the clinician should add "Late Note" to the reason for consultation
    o Example : Convulsion-2409 // Late Note
  • For Nurses: When a nurse needs to create a note when he/she on call. the nurse should add “Nurse On Call” to the reason for consultation.
    o Example : Convulsion-2409 // Nurse On Call

[2] Select from list:

  • The Consultation Reason can be document by using the “Select from list” [2A].
  • For statistic purpose, as much as possible, use the content from the "Select from list" to document the reason of consultation.

[3] The [+] button:

  • When pressing the [+] button, the “Reason for Visit: Note” Popup window appear and the user can add a note [A].
  • By pressing on “Submit” [B], the content will be added with the name of the user who made the addition and will be timestamp [C].

IMPORTANT:
  • Using the [+] button is the equivalent of an annotation in the margin of the paper file.
  • The [+] button, can also be used to clarify or add to a note already approved, whatever if the note was written by the user itself or by another user.
  • If the note is printed, the content of the “Reason for Visit: Note” [C] will not be on the printout.
